# Rust Vector Addition with HIP (FFI)

This project demonstrates how to perform a vector addition operation on an AMD GPU using HIP, with the host code written in Rust and the GPU kernel in C++ (HIP). It utilizes Rust's Foreign Function Interface (FFI) to call HIP runtime APIs and launch a pre-compiled C++ HIP kernel.

## Prerequisites

1.  **ROCm SDK**: Ensure the ROCm SDK is installed and configured correctly on your system. This provides the HIP runtime, compiler (`hipcc`), and necessary libraries.
    *   Typically, ROCm is installed in `/opt/rocm`.
2.  **Rust**: Install Rust and Cargo. You can find instructions at [https://www.rust-lang.org/tools/install](https://www.rust-lang.org/tools/install).
3.  **HIP C++ Compiler (`hipcc`)**: This is part of the ROCm SDK and should be in your PATH.
4.  **Clang (for `bindgen`)**: `bindgen` requires `libclang` to parse C/C++ headers. Ensure Clang is installed. On Ubuntu, you might install it with `sudo apt install libclang-dev clang`.

## Build and Run Instructions

### 1. Compile the C++ HIP Kernel

The GPU kernel is in `vectorAdd_kernel.hip`. It needs to be compiled into a Code Object (`.co`) file.

Navigate to the `rust_vector_add` project directory:
```bash
cd path/to/your/rust_vector_add
```

Compile the kernel using `hipcc`:
```bash
hipcc --genco vectorAdd_kernel.hip -o vectorAdd_kernel.co
```
This command creates `vectorAdd_kernel.co` in the current directory. The Rust application expects this file to be present in the working directory from which it's run (e.g., the project root when using `cargo run`).

### 2. Build the Rust Application

With the `vectorAdd_kernel.co` file in place, you can build the Rust host application:
```bash
cargo build
```
For a release build (recommended for performance):
```bash
cargo build --release
```

### 3. Run the Application

Execute the compiled Rust binary. If you built in release mode:
```bash
./target/release/rust_vector_add
```
Or, you can run directly with Cargo (this will also build if necessary):
```bash
cargo run 
```
Or for release:
```bash
cargo run --release
```
Ensure `vectorAdd_kernel.co` is in the root of the `rust_vector_add` directory if running with `cargo run`, as this is where the application will look for it by default (`./vectorAdd_kernel.co`).

## Project Structure

- `Cargo.toml`: Project manifest, includes dependencies like `anyhow`, `log`, `env_logger`, `libc`, `scopeguard`, and `bindgen` (build-dependency).
- `build.rs`: Build script that uses `bindgen` to generate Rust bindings for HIP API functions listed in `wrapper.h`.
- `wrapper.h`: C++ header file that includes necessary HIP headers for `bindgen`.
- `vectorAdd_kernel.hip`: The C++ HIP kernel code for vector addition.
- `vectorAdd_kernel.co`: (Generated by user) The compiled HIP kernel object file.
- `src/`:
    - `main.rs`: Main Rust application logic: initializes data, manages GPU memory, loads and launches the kernel, and verifies results.
    - `hip_utils.rs`: Safe Rust wrappers around HIP API calls and the `hip_call!` macro for error handling. Bindings generated by `bindgen` are included here.
- `README.md`: This file.

## How it Works

1.  **Kernel Compilation**: The `vectorAdd_kernel.hip` C++ file is compiled separately using `hipcc` into a code object (`.co`) file. This file contains the GPU executable code.
2.  **FFI Bindings**: The `build.rs` script uses the `bindgen` crate to generate Rust functions that can call into the HIP runtime library (`libamdhip64.so`). It reads `wrapper.h` to know which functions and types to generate bindings for.
3.  **Application Logic (`main.rs`)**:
    *   Initializes vectors `A` and `B` on the host (CPU).
    *   Allocates memory on the GPU for vectors `A`, `B`, and the result `C` (`hipMalloc`).
    *   Copies input vectors `A` and `B` from host memory to GPU memory (`hipMemcpy HostToDevice`).
    *   Loads the compiled kernel (`vectorAdd_kernel.co`) into GPU memory using `hipModuleLoadData`.
    *   Retrieves a handle to the `vectorAdd` kernel function within the loaded module using `hipModuleGetFunction`.
    *   Sets up parameters for the kernel launch (pointers to GPU memory for `A`, `B`, `C`, and the vector size `N`).
    *   Launches the kernel on the GPU using `hipModuleLaunchKernel`. This executes the `C[i] = A[i] + B[i]` operation in parallel across many GPU threads.
    *   Synchronizes with the GPU to ensure kernel completion (`hipDeviceSynchronize`).
    *   Copies the result vector `C` from GPU memory back to host memory (`hipMemcpy DeviceToHost`).
    *   Performs a CPU-based vector addition for verification.
    *   Compares the GPU result with the CPU result.
    *   Cleans up all allocated GPU resources (`hipFree`, `hipModuleUnload`) using `scopeguard` to ensure cleanup even in case of errors.
4.  **Error Handling**: The `hip_call!` macro in `hip_utils.rs` wraps each HIP API call. It checks the `hipError_t` return code and converts any error into an `anyhow::Result`, providing a descriptive error message.
